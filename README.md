# DGmini, a minimal 1D discontinuous Galerkin solver

A lightweight, self-contained C++ implementation of a 1D discontinuous Galerkin solver for experimentation, testing, and understanding of core DG building blocks.

---

## Dependencies

DGmini depends on the following external libraries:
- [`CMake`](https://cmake.org/) (version 3.16 or newer)
- [`yaml-cpp`](https://github.com/jbeder/yaml-cpp) for parsing YAML solver input files 
- [`muparser`](https://github.com/beltoforion/muparser) for parsing mathematical expressions

On Ubuntu, install with:
```bash
sudo apt-get install cmake libyaml-cpp-dev libmuparser-dev
```

---

## Build & Run

Manually build and run with:

```bash
cmake -S . -B build
cmake --build build

./build/dgmini --config path/to/input.yaml # To run the solver
./build/dgmini # runs with input.yaml in root

./build/dgmini_unittest # To run unittests

pytest tests/regression -v    # run regression tests
pytest tests/verification -v  # run verification tests
pytest tests -v               # run all Python-based tests
```
**VS Code**: 
The repository includes predefined tasks in `.vscode/tasks.json` for configuring CMake,
building the executables, and running unit, regression, and verification tests. Open the project in VS Code and use `Tasks: Run Task` to access them.

## Current Features:

- 1D implementation
- Modal basis functions for any order $p$
- Gauss-Legendre quadrature rule
- Convective numerical fluxes: Lax-Friedrichs, Rusanov
- Time integration: Forward Euler, Runge-Kutta RK2 / SSP RK3 / RK4
- Time step control: fixed time step size or CFL number
- PDEs: 
  - linear advection equation
  - Burgers' equation

## Project Structure

```plain
├──source/
│  ├──Algebra/      basic linear algebra (Vec, Mat, BLAS-like operations)
│  ├──FEM/          finite element building blocks (basis, quadrature)
│  ├──Mesh/         mesh and elements
│  ├──PDE/          mathematical models (convective/diffusive fluxes, sources)
│  ├──Spatial/      spatial discretization (numerical fluxes, RHS assembly)
│  ├──Temporal/     time integration (e.g. forward Euler, Runge-Kutta)
│  ├──Solver/       solver control and runtime object factory
│  ├──IO/           input/output utilities (parsing, data export)
│  └──main.cpp      program entry point
│ 
├──tests/ 
│  ├──regression/   regression tests comparing current outputs against stored reference data
│  ├──unit/         unit tests for individual components
│  └──verification/ verification tests checking expected numerical properties (e.g. convergence orders)
│
├──scripts/         helper scripts (e.g. solution visualization, convergence studies)
│
├──output/          solver output data [generated]
│
├──build/           build artifacts generated by CMake [generated]
│
└──template/        templates for new classes
```

## Architecture Overview
- *PDE* defines the physical model (e.g. linear advection, Burgers' equation)
- *Spatial* implements the spatial discretization (DG operators, numerical fluxes)
- *Temporal* handles time integration, separated from spatial discretization
- *FEM* and *Mesh* define the discrete space and geometry
- *Solver* contains the high-level solver logic: it builds runtime objects from the input configuration, initializes the solution, drives the time-stepping loop, and triggers output
- *Algebra* provides low-level linear algebra utilities
- *IO* handles input parsing and output generation

This separation keeps the code modular and allows independent extension of:
- mathematical models
- numerical schemes
- basis functions and numerical quadrature [hard-coded for now]
